## 关于对话模型的Message(消息)
聊天模型，出了将字符串作为输入外，还可以使用 聊天消息 作为输入，并返回 聊天消息 作为输出。
LangChain有一些内置的消息类型：
* 🔥 SystemMessage ：设定AI行为规则或背景信息。比如设定AI的初始状态、行为模式或对话的总体目标。比如“作为一个代码专家”，或者“返回json格式”。通常作为输入消息序列中的第一个传递。
* 🔥 HumanMessage ：表示来自用户输入。比如“实现 一个快速排序方法”
* 🔥 AIMessage ：存储AI回复的内容。这可以是文本，也可以是调用工具的请求
* ChatMessage ：可以自定义角色的通用消息类型
* FunctionMessage/ToolMessage ：函数调用/工具消息，用于函数调用结果的消息类型

    快速排序是一种高效的排序算法，使用**分治法**策略。
    
    这里我提供两种实现方式：
    1.  **Python 风格（易于理解）**：代码简洁，适合理解算法逻辑，但需要额外的 O(n) 空间。
    2.  **标准原地排序**：面试和工程中最常用的方式，空间复杂度为 O(log n)。
    
    ### 方法一：Python 风格（列表推导式）
    
    这种方式代码量最少，非常直观地展示了“分治”的思想。
    
    ```python
    def quick_sort(arr):
        # 递归终止条件：如果数组为空或只有一个元素，则直接返回
        if len(arr) <= 1:
            return arr
        
        # 选取基准值，这里选择中间的元素
        pivot = arr[len(arr) // 2]
        
        # 分割数组
        left = [x for x in arr if x < pivot]   # 小于基准的元素
        middle = [x for x in arr if x == pivot] # 等于基准的元素
        right = [x for x in arr if x > pivot]  # 大于基准的元素
        
        # 递归并合并
        return quick_sort(left) + middle + quick_sort(right)
    
    # 测试
    if __name__ == "__main__":
        data = [3, 6, 8, 10, 1, 2, 1]
        print("原始数组:", data)
        print("排序后:", quick_sort(data))
    ```
    
    ---
    
    ### 方法二：标准原地排序
    
    这是教科书和标准库中常见的实现方式。它不需要创建新的列表，直接在原数组上交换元素，空间效率更高。
    
    **核心逻辑：**
    1.  选取一个基准值。
    2.  通过 `partition` 函数将小于基准值的元素放到左边，大于的放到右边。
    3.  递归处理左右两边的子数组。
    
    ```python
    def partition(arr, low, high):
        """
        分区函数：
        将小于 pivot 的元素移到左边，大于 pivot 的移到右边。
        返回 pivot 的最终索引位置。
        """
        pivot = arr[high]  # 选择最后一个元素作为基准
        i = low - 1        # i 指向小于 pivot 的区域的最后一个位置
        
        for j in range(low, high):
            # 如果当前元素小于基准值
            if arr[j] < pivot:
                i += 1
                # 交换 arr[i] 和 arr[j]
                arr[i], arr[j] = arr[j], arr[i]
        
        # 最后将 pivot 放到正确的位置 (i + 1)
        arr[i + 1], arr[high] = arr[high], arr[i + 1]
        return i + 1
    
    def quick_sort_inplace(arr, low, high):
        """
        快速排序递归函数
        """
        if low < high:
            # pi 是分区索引，arr[pi] 已经在正确位置
            pi = partition(arr, low, high)
            
            # 递归排序基准左侧和右侧的子数组
            quick_sort_inplace(arr, low, pi - 1)
            quick_sort_inplace(arr, pi + 1, high)
    
    # 封装一层，方便调用
    def quick_sort_standard(arr):
        quick_sort_inplace(arr, 0, len(arr) - 1)
        return arr
    
    # 测试
    if __name__ == "__main__":
        data = [10, 7, 8, 9, 1, 5]
        print("原始数组:", data)
        quick_sort_standard(data)
        print("排序后:", data)
    ```
    
    ### 算法复杂度分析
    
    *   **时间复杂度**：
        *   **平均情况**：O(n log n) —— 这是最常见的情况。
        *   **最坏情况**：O(n²) —— 当数组已经是正序或逆序，且每次基准值都选到最大或最小值时。可以通过随机选择基准值来避免这种情况。
    *   **空间复杂度**：
        *   方法一：O(n) —— 因为创建了新列表。
        *   方法二：O(log n) —— 主要是递归调用栈消耗的空间。
    *   **稳定性**：快速排序是**不稳定**的排序算法（相同的元素在排序后相对位置可能改变）。

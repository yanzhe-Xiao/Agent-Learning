## 角度2：参数位置不同

### 以对话模型为例


```python
import os
from langchain_openai import  ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

chat_model = ChatOpenAI(
    # 必须设置的三个参数
    model=os.getenv("MODELSPACE_MODEL"), # 默认是gpt-3.5-turbo
    api_key=os.getenv("MODELSPACE_API_KEY"),
    base_url=os.getenv("MODELSPACE_BASE_URL")
)

response = chat_model.invoke("什么是langchain？")

print(response.content)

```

    简单来说，**LangChain 是一个开源框架，专门用于帮助开发者快速构建基于大语言模型（LLM，如 GPT-4、Llama 等）的应用程序。**
    
    如果把大语言模型（LLM）比作**“大脑”**，那么 LangChain 就是让大脑能够连接四肢、记忆和工具的**“躯干和神经系统”**。
    
    以下是关于 LangChain 的详细解读：
    
    ### 1. 为什么要由 LangChain？（解决什么痛点）
    直接调用 OpenAI 或其他模型的 API 虽然简单，但开发复杂的应用时会遇到很多问题：
    *   **数据隔离**：模型不知道你的私有数据（比如公司文档、PDF）。
    *   **没有记忆**：模型默认记不住上一句话说了什么。
    *   **无法联网或操作工具**：模型只能生成文本，不能直接查天气或操作数据库。
    
    LangChain 就是为了解决这些“最后一公里”的问题而诞生的，它提供了一套标准化的接口和组件。
    
    ### 2. LangChain 的六大核心模块
    
    LangChain 把构建 AI 应用的过程模块化了，主要包含以下几个部分：
    
    1.  **Models (模型 I/O)**：
        *   这是对各种大模型（OpenAI, Anthropic, HuggingFace 等）的统一封装。你可以很方便地从 GPT-3.5 切换到 Claude，而不需要改太多代码。
    
    2.  **Prompts (提示词管理)**：
        *   管理和优化输入给模型的指令。比如“模板化”提示词，或者“少样本提示”，让模型回答更精准。
    
    3.  **Memory (记忆)**：
        *   让 AI “记住”之前的对话内容。LangChain 提供了多种记忆组件，比如简单的缓冲记忆、窗口记忆，或者将对话存入数据库。
    
    4.  **Indexes (索引/数据检索 - RAG 的核心)**：
        *   这是 LangChain 最强大的功能之一。它允许你加载外部文档（PDF、网页、数据库），将其切片、向量化并存入向量数据库。当用户提问时，它会自动检索相关内容喂给模型。这就是著名的 **RAG（检索增强生成）** 架构。
    
    5.  **Chains (链)**：
        *   把多个操作串联起来。例如：先从用户输入中提取关键词 -> 搜索数据库 -> 将结果喂给 LLM -> 生成回答。这就是一条“链”。
    
    6.  **Agents (智能体)**：
        *   这是最先进的部分。Agent 让 LLM 变成一个“决策者”。模型会根据用户的请求，自己决定该调用什么工具（如搜索谷歌、查天气、写代码），并执行任务。
    
    ### 3. 一个通俗的例子
    
    假设你想做一个**“智能法律助手”**：
    
    *   **没有 LangChain**：你需要自己写代码去读取法律文档，处理分词，调用 OpenAI API，手动把历史对话拼接起来发给模型，还要处理超时和报错。
    *   **使用 LangChain**：
        *   用 **Document Loaders** 加载法律文档。
        *   用 **Vector Store** 把文档变成向量存储。
        *   用 **Memory** 组件自动维护对话历史。
        *   用 **Retrieval Chain** 自动查找相关法律条文并回答。
    
    ### 4. 总结
    
    **LangChain 是 LLM 应用开发的“胶水”和“脚手架”。**
    
    *   **优点**：大大降低了开发门槛，组件丰富，生态完善（支持 Python 和 JavaScript）。
    *   **缺点**：抽象层较多，有时调试比较困难；更新迭代极快，部分旧代码容易失效。
    
    如果你想开发一个不仅是简单的“问答机器人”，而是能“查阅文档、联网搜索、有记忆、能办事”的 AI 应用，LangChain 目前是首选工具。
